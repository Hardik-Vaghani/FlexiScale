# Contributing to FlexiScale

Thank you for your interest in contributing to FlexiScale! We welcome contributions from the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/flexiscale.git`
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Run tests: `./gradlew test`
6. Commit and push
7. Open a Pull Request

## Development Setup

- **JDK 21+** (resolved automatically via Foojay Toolchains plugin)
- **Android SDK** (for compose and resources modules)
- No manual Gradle installation needed — use `./gradlew`

## Code Style

- Follow Kotlin coding conventions
- Use 4-space indentation
- Add KDoc comments to all public APIs
- Keep classes focused and modular

## Pull Request Process

1. Ensure all tests pass
2. Update documentation if adding new features
3. Add or update tests for changes
4. Keep PRs focused on a single concern

## Reporting Issues

- Use the issue templates when reporting bugs or suggesting features
- Provide clear reproduction steps for bugs
- Include relevant environment details (OS, JDK version, etc.)

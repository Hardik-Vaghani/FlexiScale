Project Name:
FlexiScale

Group:
io.github.hardikvaghani.flexiscale

Author:
Hardik Vaghani

---

## PROJECT VISION

FlexiScale is a responsive UI ecosystem for Android.

The project supports:

1. XML Based UI
2. Jetpack Compose UI
3. Phones
4. Foldables
5. Tablets
6. Desktop / ChromeOS
7. TV
8. Wear OS

The project is designed as a multi-module library ecosystem.

## MODULE: flexiscale-runtime

Purpose:

Provides runtime scaling engine.

Contains:

* ScreenBucket
* ScaleProfile
* ScaleStrategy
* ScreenBucketResolver
* Responsive Scaling Logic

Used By:

* flexiscale-compose
* flexiscale-generator

Published:

YES

Artifact:

io.github.hardikvaghani:flexiscale-runtime

Consumer Access:

YES

---

## MODULE: flexiscale-generator

Purpose:

Internal development tool.

Responsible for generating XML dimension resources.

Generates:

values-swXXXdp/
dimensions.xml

Used By:

Project Maintainer Only

Published:

NO

Artifact:

NONE

Consumer Access:

NO

Notes:

This module exists only to create resource files.

Users should never depend on this module.

Generator output must be committed into
flexiscale-tokens module.

---

## IMPORTANT RULES

Rule 1

Generator never contains Android resources.

Rule 2

Tokens module is the only XML resource module.

Rule 3

Runtime never depends on Compose.

Rule 4

Compose depends on Runtime.

Rule 5

Generator depends on Runtime.

Rule 6

Consumers never use Generator.

Rule 7

All XML dimensions are generated automatically.

Rule 8

Never manually edit generated XML files.

Always regenerate them.

Rule 9

Generator is a developer tool.

Tokens are a consumer library.

Rule 10

Keep Runtime platform-independent as much as possible.
